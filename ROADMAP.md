# Software Engineer Roadmap — Python / Data / Applied AI

## Goal

Become a junior Software Engineer with strong foundations in Python, automation, data, APIs, and applied AI.

This roadmap is the stable reference for the learning path.  
Detailed progress is tracked in `progress-tracker.md`.

---

## Current Phase

**Phase 1 — Python Foundations**

---

## Roadmap Overview

| Status | Phase | Main Goal | Expected Proof |
|---|---|---|---|
| ✅ | Phase 0 — Developer Setup | Be functional with Linux, Python, Git, GitHub, and VS Code | Initial project created, venv configured, Git/GitHub workflow validated |
| 🔄 | Phase 1 — Python Foundations | Master the Python basics required for scripts and projects | Python exercises and small clean scripts |
| ⬜ | Phase 2 — Python Project, CLI, Tests | Structure a real Python project with CLI and automated tests | Tested CLI mini-project |
| ⬜ | Phase 3 — SQL and Data Quality | Query, clean, and validate data | Data cleaning project and SQL queries |
| ⬜ | Phase 4 — APIs and Backend Basics | Consume and build simple APIs | API client and small FastAPI backend |
| ⬜ | Phase 5 — Automation | Automate repetitive tasks with Python | Useful automation scripts |
| ⬜ | Phase 6 — Applied AI | Use AI inside practical applications | Simple AI assistant or AI workflow |
| ⬜ | Phase 7 — Junior Portfolio | Turn the best projects into visible proof of skills | Clean repositories, strong READMEs, clear demos |

---

## Mastery Levels

| Level | Meaning |
|---|---|
| Discovery | I understand the general idea |
| Guided Usage | I can do it with help or a template |
| Autonomy | I can do it again by myself in a new context |
| Practical Mastery | I can use it in a real project and explain my choices |

---

## Validation Rule

A phase is validated only when it produces concrete proof:

- working code
- completed exercises
- mini-project
- clear README
- clean Git commit
- push to GitHub
- ability to explain what was done

---

## Working Principles

- Move phase by phase
- Do not skip foundations
- Produce visible proof
- Commit regularly
- Keep useful public notes
- Never push secrets or sensitive information
