# Profil GitHub et repos épinglés

## Objectif
Présenter clairement mon travail et mettre en avant les projets qui prouvent mes compétences.

## Pourquoi c’est important
Un recruteur ou mentor peut regarder mon GitHub pour voir mon niveau réel : projets, README, historique, structure, tests, régularité.

## Points clés
- Avoir un profil clair.
- Épingler seulement les repos utiles.
- Mettre en avant les projets exécutables.
- Éviter d’épingler des repos brouillons.
- Choisir des projets qui montrent des compétences différentes.
- Avoir des README propres sur les repos épinglés.

## Profil GitHub
Ce qu’il doit montrer :
- nom ou pseudo clair ;
- courte description ;
- objectif professionnel ;
- stack principale ;
- lien vers portfolio ou repo hub si utile.

## Repos épinglés
Choisir les repos qui prouvent vraiment :
- Python ;
- Git ;
- CLI ;
- tests ;
- documentation ;
- backend/API plus tard ;
- SQL plus tard ;
- Docker plus tard.

## Erreurs à éviter
- Épingler trop de projets faibles.
- Épingler des repos sans README.
- Épingler des projets non exécutables.
- Mélanger exercices brouillons et projets portfolio.
- Croire que quantité > qualité.

## Résumé personnel
Mon profil GitHub doit être une vitrine claire. Les repos épinglés doivent montrer mes meilleurs projets, pas tout ce que j’ai fait.
